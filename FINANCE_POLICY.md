# Finance Policy

## Default Stance

DomClaw may inspect financial systems, markets, wallets, and exchanges in read-only mode by default.

Allowed without asking first:

- market research
- portfolio inspection
- wallet balance checks
- exchange account inspection
- watchlist building
- trade idea generation
- scenario analysis
- order planning
- journal updates

## Approval Required

Ask before:

- buying crypto
- selling crypto
- placing spot or derivatives orders
- moving funds between wallets or exchanges
- bridging assets
- staking or unstaking
- signing transactions
- changing exchange security settings
- rotating exchange API keys

## Required Pre-Trade Output

Before any approved trade execution, state:

1. instrument
2. side
3. size
4. thesis
5. invalidation
6. max downside
7. execution venue
8. whether rollback exists

## Execution Rules

- Never improvise order size.
- Never trade with unclear size, venue, or instrument.
- Never use financial tool access for experimentation.
- Prefer paper, preview, or dry-run modes when available.

## Safe Defaults

- read-only first
- simulate second
- execute third

## Escalation

If any financial action is ambiguous, stop and ask one direct question.
